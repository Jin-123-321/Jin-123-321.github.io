泰坦尼克号生还预测 — 数据清洗到模型调优

发表于2025-03-28|更新于2026-05-22|项目展示机器学习
|浏览量:

项目名片

维度 内容
业务问题 根据乘客属性预测其是否在泰坦尼克号沉船中生还
数据集 Kaggle Titanic，训练集 891 条 × 12 特征，测试集 418 条
核心模型 随机森林（Random Forest）
关键指标 准确率 83.8%（调优后），较基线提升 1.3 个百分点
核心工具 Python · Scikit-learn · Pandas · Matplotlib · Seaborn
项目链接 GitHub Repository

核心成果

本项目的最终模型在 Kaggle 测试集上取得了 83.8% 的预测准确率，优于未调优的基线模型（82.5%）。

随机森林天生对缺失值和异常值鲁棒，不需要复杂的数据标准化，是一个非常适合表格数据的”开箱即用”模型。通过 5 折交叉验证的 GridSearchCV，我们找到了一组兼顾精度与泛化能力的最优参数。

调优过程中一个有趣的发现是：限制树的深度反而提升了泛化性能——max_depth=6 的约束避免了单棵决策树过度拟合训练集中的噪声模式。

方案设计

数据清洗

数据集存在三类缺失值：

  • Cabin（77% 缺失）：根据舱位等级（Pclass）推断是否有舱房记录，构造 HasCabin 二进制特征
  • Age（20% 缺失）：按 Pclass + Sex 分组中位数填充，保留不同人群的年龄分布差异
  • Embarked（2 条缺失）：众数填充

特征工程

从 12 个原始特征出发，构造了以下高价值特征：

  • Title：从乘客姓名中提取称谓（Mr/Mrs/Miss 等），与性别、社会地位强相关
  • FamilySize：SibSp + Parch + 1，家庭规模——独行旅客生还率显著低于 2-4 人家庭
  • IsAlone：标记独行旅客
  • AgeGroup：将年龄离散化为儿童/青年/中年/老年四组
  • FareBin：将票价按分位数分组，避免极端高价票的杠杆效应

最终保留 11 个特征进入模型，去除了高缺失率和冗余字段。

模型选择与调优

选用随机森林作为核心模型，理由有三：对缺失值鲁棒、天然支持特征重要性排序、无需特征缩放。

通过 GridSearchCV 进行 5 折交叉验证的超参数搜索：

参数 搜索空间 最优值
n_estimators 100, 200, 300 200
max_depth 4, 6, 8, None 6
min_samples_split 2, 5, 10 2
min_samples_leaf 1, 2, 4 1

核心代码实现

特征重要性

Title（称谓）以超过 20% 的重要性位列第一，验证了”女士与儿童优先”的历史事实在数据中的体现。其次为 Age 和 Fare——年龄和票价真实反映了乘客的社会阶层与逃生优先级。

关键发现

  1. 社会阶层是生还的最强信号：头等舱（Pclass=1）生还率约 63%，远高于三等舱的 24%，这一差异不仅体现在物理位置（靠近救生艇），也与乘客的社会经济地位直接相关
  2. 家庭结构的非线性效应：独行旅客生还率仅 30%，2-4 人家庭的生还率达 55%，但大型家庭（5+人）的生还率又下降到 20%——可能在危机中互相等待而延误逃生
  3. 性别差异极为显著：女性生还率约 74%，男性仅 19%，这一差异在大部分年龄组和舱位等级中都稳定存在，是数据中最重要的单一预测信号

技术栈

Python · Pandas · Scikit-learn · Matplotlib · Seaborn
