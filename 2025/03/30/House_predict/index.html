房价预测 — 从数据清洗到多模型对比

发表于2025-03-30|更新于2026-05-22|项目展示机器学习
|浏览量:

项目名片

维度 内容
业务问题 基于 80+ 房屋特征预测最终售价
数据集 Kaggle House Prices，训练集 1,460 条 × 81 特征，测试集 1,459 条
核心模型 随机森林（最优）、XGBoost、线性回归
关键指标 R² = 0.859，RMSE = $28,875（随机森林）
核心工具 Python · Scikit-learn · XGBoost · Matplotlib · Seaborn
项目链接 GitHub Repository

核心成果

经过三种模型的系统对比，随机森林以 R² = 0.8587 和 RMSE $28,875 成为最优模型，能够解释约 86% 的房价变动，平均预测误差不到三万美元。

这个结果特别值得关注的是：随机森林仅使用了 9 个核心特征就达到了这个性能。OverallQual（整体质量评分）以 56.9% 的特征重要性压倒性地主导了预测——这意味着房子的”综合品质感”比任何单一物理指标（面积、房间数）都更能解释价格。

XGBoost（R² = 0.8537）在这个数据集上并未超越随机森林，分析原因：1,460 条的训练样本对于梯度提升来说偏小，容易过拟合；而随机森林的 Bagging 策略在小样本下天然更稳健。

方案设计

数据准备

数据集有 81 个原始特征，但许多字段缺失率极高（如泳池质量 PoolQC 缺失 99.5%）。处理策略：

  • 类别型高缺失特征（PoolQC、Fence、Alley 等）：填充 “None”，表示该房屋无此设施
  • LotFrontage（街道连接长度）：按 Neighborhood 分组中位数填充——同一社区的街道宽度通常一致
  • GarageYrBlt（车库建造年份）：无车库的房屋填充 0
  • MasVnrArea（石材饰面面积）：无石材的填充 0

从 EDA 到特征选择

目标变量 SalePrice 呈明显右偏分布（偏度 ≈ 1.88），经 log1p 变换后近似正态——这是回归模型的关键前提：

通过 Pearson 相关系数筛选与房价最相关的 Top 20 特征：

最终选定 9 个兼顾解释力与低共线性的核心特征：OverallQual、GrLivArea、GarageCars、TotalBsmtSF、FullBath、YearBuilt、YearRemodAdd、Fireplaces、MasVnrArea。

三模型对比

模型 RMSE（log 空间） R² RMSE（原始房价）
线性回归 0.1685 0.8478 $31,646
随机森林 0.1624 0.8587 $28,875
XGBoost 0.1652 0.8537 $28,852

线性回归作为基线已经表现不错（R²=0.85），说明核心特征与房价之间确实存在较强的线性关系。随机森林在此基础上将误差降低了约 9%，主要得益于捕捉了特征之间的非线性交互（如质量 × 面积）。

核心代码实现

关键发现

  1. OverallQual 是绝对王者：56.9% 的特征重要性说明，在房地产评估中，综合品质评分比任何单一物理指标都更重要。房屋是一个”整体产品”，而非各部分的简单加总
  2. 面积不是越大越好：GrLivArea（地上居住面积）以 15.9% 排名第二，但与 OverallQual 之间存在显著的交互效应——低品质的大房子不如高品质的中等户型值钱
  3. 小样本下 RF > XGBoost：在 1,460 条数据上，XGBoost 的梯度提升策略并未带来预料中的优势，Bagging 的方差缩减在此场景下更为有效

技术栈

Python · Pandas · Scikit-learn · XGBoost · Matplotlib · Seaborn
