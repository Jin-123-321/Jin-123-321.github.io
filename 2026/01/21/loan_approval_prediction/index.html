贷款审批预测 — 多模型对比与 SHAP 可解释性分析

发表于2026-01-21|更新于2026-05-22|项目展示机器学习
|浏览量:

项目名片

维度 内容
业务问题 基于申请人收入、信用和资产信息预测贷款是否应获批
数据集 FinTech 信贷数据，4,269 条 × 12 特征，正例占比 62%
核心模型 LightGBM（最终推荐）、随机森林、逻辑回归、神经网络
关键指标 AUC 99.9%，仅 1 个误批 + 0 个误拒（LightGBM）
核心工具 Python · Scikit-learn · LightGBM · SHAP · Matplotlib · Seaborn
项目链接 GitHub Repository

核心成果

LightGBM 以 AUC 99.9% 的接近完美表现被选为最优部署模型。在全部 854 条测试样本中，仅产生 1 个错误批准（将本应拒绝的申请误判为通过）和 0 个错误拒绝——这对于高度风险敏感的信贷场景来说是非常理想的结果。

但更重要的是：我们通过 SHAP 分析验证了模型并非”黑箱决策”。信用评分（cibil_score）在审批决策中占据绝对主导地位，其次为收入水平和资产负债比率——这一发现与传统信贷风控的核心逻辑完全一致，说明模型真正学到了符合金融直觉的风险结构，而非数据中的表面模式。

值得注意的是，随机森林在此数据集上达到了 100% 准确率，但这很可能暗示存在过拟合或数据泄露风险。相比之下，LightGBM（99.9% AUC）更为稳健，是工程部署的正确选择。

方案设计

问题定义

在 FinTech 信贷场景中，贷款审批不是简单的二分类问题，而是高度风险敏感的决策：

  • 错误批准 → 直接导致违约和资金损失
  • 错误拒绝 → 意味着潜在收益流失和客户体验下降

因此本项目不仅追求预测精度，更强调风险控制、模型稳定性和可解释性三者的平衡。

双路径特征工程

针对不同模型族的特性，设计了两条互补的特征工程路径：

路径 方法 适用模型 特点
PCA 降维 保留 95% 方差，12 维 → 9 维 逻辑回归、MLP 消除共线性，提升线性模型稳定性
业务特征构造 5 个金融含义特征 随机森林、LightGBM 增强树模型对业务逻辑的理解

构造的 5 个业务特征：

特征 公式 业务意义
total_assets 各类资产求和 总资产水平
income_to_loan_ratio 收入 / 贷款金额 还款能力（DTI 的反向）
asset_to_loan_ratio 总资产 / 贷款金额 风险缓冲
asset_to_income_ratio 总资产 / 收入 财务健康度
income_per_dependent 收入 / (赡养人数+1) 人均可支配收入

四模型对比

模型 Accuracy F1 ROC-AUC 误批 误拒
LightGBM（推荐） 0.999 0.999 1.000 1 0
随机森林 1.000 1.000 1.000 0 0
逻辑回归（PCA） 0.920 0.936 0.973 47 21
神经网络（FE） 0.917 0.934 0.977 33 38

核心代码实现

SHAP 可解释性

SHAP 分析揭示的决策逻辑：

  • cibil_score 在审批决策中占据绝对主导，高信用分直接对应批准——这完全符合信贷风控的基本原则
  • income_annum 和 income_to_loan_ratio 对审批有显著正向影响——还款能力是除信用历史外最重要的考量
  • loan_amount 对高额度申请有抑制作用——体现了风险敞口控制的逻辑
  • asset_to_loan_ratio 提供了有效的风险缓冲信号——资产充足的申请人获得更多信任

关键发现

  1. 信用评分是核心中的核心：SHAP 分析中 cibil_score 的影响力远超其他特征，验证了传统风控的核心逻辑在机器学习时代依然成立
  2. 业务特征 > 降维特征：采用业务特征工程的神经网络（AUC 0.977）显著优于 PCA 降维的线性模型（AUC 0.973），说明金融领域的专家知识在特征工程中仍然不可替代
  3. 100% 准确率反而需要警惕：随机森林的完美分类更可能是过拟合信号，在真实风控系统中，99.9% AUC + 可解释的决策逻辑比 100% + 黑箱更有部署价值

技术栈

Python · Scikit-learn · LightGBM · SHAP · Matplotlib · Seaborn
